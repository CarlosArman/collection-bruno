Thank you for your interest in contributing.

This repository is focused on **educational API collections** using Bruno, with emphasis on clarity, reuse, and practical QA value.

---

## 1. Contribution Principles

Please follow these principles when contributing:

- clarity over complexity
- reusable environments and variables
- readable request and folder naming
- educational value over over-engineering
- documentation updates when behavior changes

---

## 2. Useful Contribution Types

- add new public API collections
- improve existing collections
- enhance variable usage (dynamic / runtime)
- add examples for scripts
- improve documentation
- improve reporting examples

---

## 3. Suggested Workflow

1. Fork the repository
2. Create a feature branch
3. Add or improve a collection
4. Update documentation if needed
5. Open a Pull Request with a clear explanation

---

## 4. Branch Naming Examples

```bash
feature/add-dummyjson-collection
feature/improve-booking-flows
docs/update-bruno-variables-guide
```

---

## 5. Pull Request Expectations

A good Pull Request should explain:

- what was added or changed
- why it was added
- which collection or API is affected
- whether documentation was updated

---

## 6. Best Practices

- group requests by domain (Auth, Users, Products, etc.)
- centralize base URLs in environments
- avoid hardcoding credentials
- reuse runtime variables when possible
- keep collections easy to understand
